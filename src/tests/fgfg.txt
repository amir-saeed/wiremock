
## Dependencies

### AWS Services
- **AWS Glue** — runs the ETL job
- **AWS S3** — stores the job script, wheel library, and source parquet files
- **AWS RDS PostgreSQL** — target database (`entity_store.quote`)
- **AWS Glue Catalog** — metadata store; the job reads source data via `spark.table()` not raw S3
- **AWS IAM** — authentication to RDS (no passwords stored; IAM token generated at runtime)

### S3 Buckets
| Purpose | Bucket |
|---------|--------|
| Glue scripts + wheel | `b4c-r1c3{env}-eqd-glue-scripts` |
| Source parquet (quotes) | `b4c-dal-{env}-data-quotes-shredded-lfs` |

### Glue Catalog
- Database: `dal_{env}_qj_db`
- Table: `b4c_request_quote`
- Partition keys: `dal_year`, `dal_month`, `dal_day`

### PostgreSQL
- Schema: `entity_store`
- Table: `entity_store.quote`
- Partitioned monthly by `reporting_date` (e.g. `quote_2026_06`)
- The Glue user must have `SELECT, INSERT, UPDATE, DELETE` on `entity_store.quote` and `USAGE` on schema `entity_store`
